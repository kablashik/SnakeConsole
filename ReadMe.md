В рамках прошлого задания у нас уже получилась полноценная игра. 
Но метод Main выглядит очень загруженным. Логично будет вынести всю ответственность за процесс игры в отдельный класс **GameController**, который будет контролировать все возможные игровые события (случай проигрыша, подбор еды змейкой, нажатие клавиши пользователем):

```
Пвевдокод:

класс GameController
{
	метод [bool] Update(ConsoleKey consoleKey)
	{
		если (игрок проиграл)
		{
			Возвращаем true
		}

    если (змейка подобрала еду)
		{
			Увеличиваем размер змейки
			Генерируем новую еду
		}

		если (нажата кнопка на клавиатуре)
		{
			Меняем направление змейке
		}

		иначе
		{
			Двигаем змейку в прежнем направлении
		}		
		
		Возвращаем false // До этой строки дойдем в случае, если игрок не проиграл.				
}
```

При этом метод Main в бесконечном цикле будет вызывать метод **Update** объекта класса GameController:

```
Псевдокод метода Main:

...
Создаем игровые объекты, в том числе объект класса GameController;

бесконечный цикл
{
	...
	если (gameController.Update(pressedKey)) // Если метод Update вернул значение True - 
	{																				 // значит игрок проиграл	
		Очищаем консоль
	  Выводим сообщение о проигрыше

		Выходим из бесконечного цикла
		...	
	} 
}
```

Обратите внимание, что в классе **InputController** появился публичный метод **MoveSnake**, куда вынесена логика по передвижению змейки. Для того, чтобы из объекта класса **GameController** можно было передвигать змейку, используя данный метод.

```
class InputController
{
	...
	public void MoveForward()
	{
	    _snake.MoveForward(); // Двигаем змейку.
	}
}
```

Ваша задача:

Дописать класс **GameController**, отвечающий за процесс игры. В рамках данного класса допишите:

1. Метод “**Update**”, который будет вызываться в каждом игровом кадре. Этот метод будет принимать клавишу, на которую пользователь нажал, а также:
    1. Проверять, проиграл ли игрок, если проиграл - возвращать true.
    2. Проверять, подобрала змейку еду, если подобрала - увеличивать змейку и генерировать новую еду.
    3. Менять направление змейке при нажатой кнопке, иначе - двигать змейку в прежнем направлении.
